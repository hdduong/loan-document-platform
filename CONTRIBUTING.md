# Contributing

Read `CLAUDE.md`, the [project constitution](.specify/memory/constitution.md), the active feature specification/plan/tasks, the OpenAPI contract, architecture, and security documentation before changing behavior. Follow the [Spec Kit workflow](docs/spec-driven-development.md) and reconcile artifacts whenever implementation changes their documented state. Authorized maintainers work on a short-lived branch, add focused tests, and submit a pull request. External contributions are not accepted until the owner adopts an explicit contribution and source license. Keep generated API types derived from OpenAPI and keep the React application inside `apps/web`.

Before review, run the repository validator, Python coverage/lint, OpenAPI validation, CloudFormation lint, and—once present—the React typecheck, per-file coverage, production build, and Playwright suite. Every production Python file must remain at least 80% line-covered individually and overall; every authored React/TypeScript production file must remain at least 80% covered per file for statements, lines, functions, and branches. Never weaken a threshold or exclusion to pass. Every pull request must request GitHub Copilot review for its current head SHA and wait for it to finish. Address every actionable, constitution-compatible comment; document why any rejected suggestion is inapplicable or harmful. Pushes made in response require another Copilot review before merge. Use only synthetic data. IDP configuration changes must update their reviewed digest and include accuracy, selection, latency, and cost regression evidence.
