# Claude Code integration

This directory contains the project-scoped Claude Code integration generated by [GitHub Spec Kit](https://github.com/github/spec-kit) v0.12.15.

## Layout

Spec Kit's Claude integration uses skills, not legacy command files. Each core command is a self-contained `SKILL.md` under `.claude/skills/`:

- `speckit-analyze`
- `speckit-checklist`
- `speckit-clarify`
- `speckit-constitution`
- `speckit-converge`
- `speckit-implement`
- `speckit-plan`
- `speckit-specify`
- `speckit-tasks`
- `speckit-taskstoissues`

Invoke a skill with a hyphenated slash command, such as `/speckit-specify` or `/speckit-plan`. Do not add duplicate `.claude/commands/speckit.*.md` files.

The skills call the PowerShell scripts and Markdown templates under `.specify/`. They must remain version-aligned with the integration metadata in `.specify/init-options.json` and `.specify/integration.json`.

## Using Claude Code

Start Claude Code at the repository root or anywhere below it. Claude Code discovers project skills in parent directories up to the repository root, so a session started in `apps/web` still receives these skills. It also loads the root `CLAUDE.md`; the nested `apps/web/CLAUDE.md` becomes active for UI files.

After first checking out a revision that creates `.claude/skills/`, restart an already-running Claude Code session if the skills are not visible. Verify the installation with `/skills` and verify instruction loading with `/memory`.

Run the feature workflow in this order unless the task calls for only a review step:

1. `/speckit-constitution`
2. `/speckit-specify`
3. `/speckit-clarify`
4. `/speckit-plan`
5. `/speckit-tasks`
6. `/speckit-analyze`
7. `/speckit-implement`
8. `/speckit-converge`

`/speckit-checklist` is available for focused requirements-quality checks. `/speckit-taskstoissues` requires an authenticated GitHub MCP server and creates GitHub issues; use it only when issue creation is explicitly part of the task.

## Repository safety

- Commit project skills, but never commit Claude credentials, tokens, local permissions, or personal settings.
- Keep personal instructions in `CLAUDE.local.md` and local Claude settings in `.claude/settings.local.json`; both must remain untracked.
- Review generated skill changes when upgrading Spec Kit. Do not modify generated `SKILL.md` files manually.
- Claude instructions guide behavior but are not a security boundary. Enforce mandatory controls in code, IAM, policies, tests, and CI.

See [Spec Kit integrations](https://github.github.com/spec-kit/reference/integrations.html) and [Claude Code skills](https://code.claude.com/docs/en/slash-commands) for the upstream behavior.
